# 检查单 (Checklist App)

一个现代化的检查单管理应用，支持多列表管理、拖拽排序、分享功能和云端同步。

![应用截图](./screenshot.png)

## ✨ 特性

### 核心功能
- 📝 **多检查单管理** - 创建和管理多个独立的检查单
- ✅ **待办项管理** - 添加、编辑、完成和删除待办项
- 🎨 **自定义主题** - 6种配色方案可选（蓝色、靛蓝、翠绿、琥珀、玫瑰、紫色）
- 🔢 **份数统计** - 为每个待办项设置份数，方便批量复制场景

### 高级功能
- 🔗 **分享功能** - 生成可分享的URL链接，支持只读查看和导入
- 📤 **导入导出** - 从分享链接导入检查单到本地
- ☁️ **云端同步** - 支持S3兼容存储（AWS S3、CloudFlare R2、MinIO等）
- 🎯 **拖拽排序** - 支持检查单卡片和待办项的拖拽重排序
- 💾 **本地存储** - 自动保存到浏览器localStorage

### 用户体验
- 📱 **响应式设计** - 完美适配桌面端和移动端
- 🌙 **优雅交互** - 流畅的动画和过渡效果
- 🎯 **直观操作** - 简洁明了的用户界面
- ⚡ **快速响应** - 无需后端，纯前端应用

## 🚀 快速开始

### 在线使用

直接在浏览器中打开 `index.html` 文件即可使用，无需安装任何依赖。

### 本地部署

1. 克隆或下载项目文件
2. 使用任意Web服务器托管文件（推荐方式）：

```bash
# 使用Python内置服务器
python3 -m http.server 8000

# 或使用Node.js的http-server
npx http-server -p 8000
```

3. 在浏览器中访问 `http://localhost:8000`

### 云端同步配置（可选）

应用支持通过S3兼容存储进行数据同步：

1. 点击左下角的设置按钮 ⚙️
2. 填入S3配置信息：
   - **Endpoint**: S3服务地址（可选，AWS S3可留空）
   - **Region**: 区域代码（如 `us-east-1`）
   - **Bucket Name**: 存储桶名称
   - **Access Key ID**: 访问密钥ID
   - **Secret Access Key**: 访问密钥

3. 点击"保存并同步"

> 🔒 所有密钥信息仅保存在本地浏览器中，不会上传到任何第三方服务器

## 📖 使用指南

### 创建检查单

1. 点击"新建检查单"按钮
2. 输入检查单名称
3. 选择一个配色主题
4. 点击"保存"

### 管理待办项

- **添加项目**: 在底部输入框输入内容，点击 ➕ 按钮
- **标记完成**: 点击待办项左侧的圆形按钮
- **调整份数**: 使用 ➕ ➖ 按钮调整份数
- **删除项目**: 点击待办项右侧的删除按钮
- **拖拽排序**: 长按待办项拖动重新排序

### 分享检查单

1. 打开要分享的检查单
2. 点击顶部的分享按钮 🔗
3. 复制生成的URL分享给他人
4. 接收者可以查看并选择导入到自己的应用中

### 拖拽排序

- **检查单排序**: 在左侧列表中拖动检查单卡片
- **待办项排序**: 在详情页面中拖动待办项

## 🔧 功能说明

### 数据存储

应用使用浏览器的 `localStorage` 存储数据，存储键为 `fileMemoData_v7`。数据格式为：

```javascript
[
  {
    id: "unique-id",
    name: "检查单名称",
    color: "blue",
    createdAt: 1700000000000,
    items: [
      {
        id: "item-id",
        text: "待办项内容",
        status: false,
        count: 1
      }
    ]
  }
]
```

### 分享机制

分享功能通过URL参数传递数据：
- 检查单数据经过Base64编码附加到URL中
- 分享链接打开时自动进入只读模式
- 用户可选择将分享的检查单导入到本地

### 云端同步

- 支持任何S3兼容的对象存储服务
- 自动上传和下载数据
- 配置信息加密存储在本地
